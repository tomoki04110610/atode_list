「知らないこと」「気になったこと」を即座にメモし、後で調べて解決するサイクルを作る知識管理アプリ。

## 1. コンセプト
- **シンプル**: 140文字以内のメモを放り込むだけ。
- **低リスク**: メールアドレス不要。ニックネームとパスワードのみで管理。
- **手放し運用**: 週1回のまとめ通知とPWAバッジによる自発的な再訪。

## 2. データベース設計 (PostgreSQL)

### users テーブル (ユーザー管理)

| カラム名 | 型 | 制約 | 用途 |
| :--- | :--- | :--- | :--- |
| nickname | string | unique, null: false, 3-15文字 | ログインID |
| password_digest | string | null: false | 暗号化パスワード |
| notification_wday | integer | default: 6 | 通知曜日 (0:日〜6:土) |
| notification_time | time | default: '20:00' | 通知時間 |

### memories テーブル (メモ本体)

| カラム名 | 型 | 制約 | 用途 |
| :--- | :--- | :--- | :--- |
| user_id | references | null: false | ユーザー紐付け |
| content | string | null: false, 最大140文字 | メモ内容 |
| status | integer | default: 0 | 0:未解決 / 1:解決済み |
| completed_at | datetime | | 「調べた！」押下日時 |

## 3. 主要ロジック・制限
- **未解決制限**: 1ユーザーにつき最大 **30件** まで（超過時は保存不可・黄色警告）。
- **履歴制限**: 1ユーザーにつき最大 **100件** まで（超過時は保存不可・赤色警告）。
- **通知**: 週1回、指定日時に未解決件数をプッシュ通知。
- **PWA**: ホーム画面アイコンに未解決件数をバッジ表示。

## 4. 画面構成
1. **トップ (Index)**: 投稿フォーム（レスポンシブ） + 未解決リスト。
2. **履歴 (History)**: 解決済みリスト（新しい順）。
3. **新規登録 (Signup)**: パスワード忘却自己責任の警告（黄色）を表示。
4. **ログイン (Login)**: 永続Cookieによる自動ログイン。
5. **設定 (Settings)**: 通知設定 + ログアウト。
